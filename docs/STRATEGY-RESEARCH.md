## ЧАСТЬ 1. ВОЕННОЕ ИСКУССТВО → БИЗНЕС-СТРАТЕГИЯ

### 1.1. Сунь Цзы — «Искусство войны» (2 500 лет)

> «Величайшая победа — та, что достигнута без боя.»

| Принцип | Военный | Бизнес-применение |
|---------|---------|-------------------|
| **Познай себя и врага** | Разведка + самоанализ | Конкурентный анализ + честная оценка своих сил |
| **Побеждай без боя** | Деморализуй врага | Создай Blue Ocean — конкуренция нерелевантна |
| **Используй местность** | Выбирай поле битвы | Выбирай рынок/нишу, где у тебя преимущество |
| **Скорость** | Быстрое развёртывание | Быстрая итерация (Lean Startup) |
| **Гибкость** | Адаптируйся к ситуации | Pivot, agile, OODA Loop |
| **Обман** | Скрывай намерения | Stealth mode, неожиданный запуск |
| **Концентрация сил** | Удар в одну точку | Beachhead market — 100% фокус на одном сегменте |

### 1.2. Клаузевиц + Наполеон (200 лет)

> «Война — продолжение политики другими средствами.» — Клаузевиц

**9 принципов войны → бизнес:**

| # | Принцип | Бизнес-аналогия |
|---|---------|----------------|
| 1 | **Цель** | Ясная миссия. Amazon: «самая клиентоцентричная компания на Земле» |
| 2 | **Наступление** | Действуй первым. Не жди — атакуй рынок |
| 3 | **Массирование** | Концентрируй ресурсы в точке прорыва |
| 4 | **Маневр** | Скорость + неожиданность + экономия усилий |
| 5 | **Единоначалие** | Один лидер, чёткая вертикаль |
| 6 | **Безопасность** | Защищай конкурентные преимущества (moat) |
| 7 | **Внезапность** | Делай то, чего конкуренты не ждут |
| 8 | **Экономия сил** | Минимум ресурсов на второстепенное |
| 9 | **Простота** | Простые планы выполняются. Сложные — нет |

**Ключевое от Наполеона:** Быстрота решений в хаосе. Не ждать идеальной информации — действовать на 70% данных.

### 1.3. OODA Loop (полковник Бойд, 1970-е)

> «Победитель — тот, кто быстрее проходит цикл решений.»

```
OBSERVE (наблюдай) → ORIENT (оценивай) → DECIDE (решай) → ACT (действуй)
         ↑                                                        |
         └────────────────────────────────────────────────────────┘
```

**Бойд «40 секунд»:** побеждал любого лётчика за 40 секунд, проходя цикл OODA быстрее противника.

**Для стартапа:** Скорость итерации = скорость OODA. Кто быстрее учится — тот побеждает.

| Этап | Стартап |
|------|---------|
| Observe | Метрики, обратная связь, рынок |
| Orient | Анализ, паттерны, контекст |
| Decide | Гипотеза, приоритизация |
| Act | MVP, тест, запуск |

## ЧАСТЬ 2. ФИЛОСОФИЯ ПОБЕДИТЕЛЕЙ

### 2.1. Стоицизм (Марк Аврелий, Сенека, Эпиктет — 2 000 лет)

> «Ты имеешь власть над своим умом — не над внешними событиями.» — Марк Аврелий

| Принцип | Значение | Применение |
|---------|---------|------------|
| **Дихотомия контроля** | Контролируй только то, что в твоей власти | Фокусируйся на действиях, не на результатах |
| **Amor Fati** | Люби судьбу | Каждая неудача = топливо для роста |
| **Premeditatio Malorum** | Предвосхищай худшее | Pre-mortem: что может убить стартап? |
| **Живи ниже средств** | Дисциплина расходов | Burn rate контроль, runway |
| **Memento Mori** | Помни о смерти | Срочность: не откладывай запуск |

**Стоики-миллиардеры:** Tim Ferriss, Tobi Lütke (Shopify CEO), Robert Rosenkranz, Jeff Bezos (цитирует Сенеку).

### 2.2. Роль удачи и случайности (Kahneman + Taleb)

> «Успех = Талант + Удача. Большой успех = Чуть больше таланта + Много удачи.» — Daniel Kahneman (Нобель)

**Научные факты:**
- Таланты распределены нормально (Гаусс), богатство — степенным законом (Power Law)
- ~50% разницы в доходах НЕ объясняются личными качествами
- Люди страдают «иллюзией контроля» — путают тайминг/удачу с заслугами

**Как «создавать удачу»:**
1. **Увеличивай «площадь удачи»** — больше контактов, больше экспериментов
2. **Держи ликвидность** — будь готов схватить возможность
3. **Барбелл (Талеб)** — 80% безопасно + 20% высокий upside
4. **Слабые связи (Грановеттер)** — случайные знакомые приносят лучшие возможности

### 2.3. Визуализация и достижение целей (нейронаука)

**Факт:** Мозг активирует те же нейронные пути при визуализации, что и при реальном действии (Nature, PubMed).

- Олимпийские атлеты используют mental rehearsal как обязательную часть подготовки
- Визуализация улучшает моторное обучение через offline-консолидацию (сон)
- Метаанализ: и кинестетическая, и визуальная имагинация активируют моторные зоны

**Практика для основателя:**
1. Визуализируй конечный результат (exit / PMF / рост) — 5 мин/день
2. Визуализируй ПРОЦЕСС, не только результат (как именно ты достигаешь)
3. «Предварительное проживание» трудных разговоров и презентаций

## ЧАСТЬ 3. СТРАТЕГИЧЕСКОЕ ПЛАНИРОВАНИЕ

### 3.1. OKR — Objectives & Key Results (Intel → Google)

> Создан Andy Grove (Intel). Внедрён John Doerr в Google. Масштабирован на Amazon, Uber, тысячи компаний.

**Формула:**
```
Objective (КУДА идём?)  → Вдохновляющая, качественная цель
Key Results (КАК узнаем, что дошли?) → 3–5 измеримых результатов
```

**Правила:**
- 60–70% достижения = хорошо (stretch goals)
- Публичные и прозрачные
- НЕ привязаны к зарплате/оценке
- Квартальный цикл

### 3.2. Ray Dalio — Principles (Bridgewater, крупнейший хедж-фонд мира)

> «Боль + Рефлексия = Прогресс»

**Система принятия решений:**
1. Решай на основе **долгосрочных** последствий (2-й и 3-й порядок), не ближайших
2. Используй **believability-weighted** голосование — мнения взвешены по опыту
3. Для каждого решения: кто уже делал это 3+ раз успешно?
4. **Radical transparency** — все ошибки обсуждаются открыто

### 3.3. Charlie Munger — Решётка ментальных моделей (Berkshire Hathaway)

> «Нельзя ничего понять, запоминая изолированные факты. Нужна решётка моделей в голове.»

**Топ-10 моделей Мангера:**

| # | Модель | Суть |
|---|--------|------|
| 1 | **Инверсия** | Спроси «что может пойти НЕ так?» вместо «как преуспеть» |
| 2 | **Круг компетенций** | Знай границы своего знания. Не выходи за них |
| 3 | **Первые принципы** | Разбирай проблему до фундамента, не полагайся на аналогии |
| 4 | **Lollapalooza** | Несколько bias'ов одновременно = непропорционально большой эффект |
| 5 | **Margin of Safety** | Всегда оставляй запас прочности |
| 6 | **Compound interest** | Сложный процент = самая мощная сила (деньги, знания, связи) |
| 7 | **Incentives** | Люди делают то, за что их вознаграждают |
| 8 | **Second-order thinking** | Думай о последствиях последствий |
| 9 | **Map ≠ Territory** | Модель ≠ реальность. Всегда сверяй |
| 10 | **Hanlon's Razor** | Не приписывай злому умыслу то, что объясняется некомпетентностью |

### 3.4. Systems Thinking — Пятая дисциплина (Peter Senge, MIT)

**5 дисциплин обучающейся организации:**
1. **Системное мышление** (связывает всё)
2. Личное мастерство
3. Ментальные модели
4. Общее видение
5. Командное обучение

**Ключевая идея:** Ищи **точки рычага** (leverage points) — места в системе, где малое усилие даёт огромный результат.

## ЧАСТЬ 4. СОЦИОЛОГИЯ УСПЕХА

### 4.1. Сила слабых связей (Грановеттер, Stanford, 1973)

> Самые важные возможности приходят от **случайных знакомых**, а не от близких друзей.

- Близкие друзья знают тех же людей и ту же информацию, что и вы
- Случайные знакомые (weak ties) имеют доступ к **другим сетям**
- Stanford + MIT + Harvard (2022): экспериментально подтверждено на масштабных данных
- ~70 000 цитирований — одна из самых влиятельных работ в социологии

**Для основателя:** Ходи на конференции, знакомься с людьми из ДРУГИХ индустрий. Лучшие идеи и связи — на стыке.

## ЧАСТЬ 5. ПРИВЛЕЧЕНИЕ ИНВЕСТИЦИЙ

### 5.1. Pitch Deck — формат Sequoia Capital

> Инвесторы тратят **~146 секунд** на просмотр deck. Первые 3 слайда определяют 70% решения. Только **1%** deck'ов получают финансирование.

**Структура (Sequoia Framework):**

| # | Слайд | Что показывает |
|---|-------|---------------|
| 1 | **Purpose** | Одно предложение: что вы делаете |
| 2 | **Problem** | Боль клиента, почему текущие решения плохи |
| 3 | **Solution** | Как вы решаете проблему |
| 4 | **Why Now** | Почему СЕЙЧАС — тайминг |
| 5 | **Market** | TAM → SAM → SOM |
| 6 | **Competition** | Конкуренция = валидация рынка |
| 7 | **Product** | Демо, скриншоты |
| 8 | **Business Model** | Как зарабатываете |
| 9 | **Team** | Почему именно вы |
| 10 | **Ask** | Сколько привлекаете и на что |

**Правила:**
- Структура > дизайн. Логика нарратива > красота слайдов
- Начинай с сильнейшего — VCs «pattern-match», а не читают последовательно
- 20–30% просматривают на мобильном
- AI-системы фильтруют deck'и ДО человека

### 5.2. Формула Fundraising

```
Traction + Team + TAM + Timing = Funding
```

- **Traction** = MRR, рост, retention
- **Team** = Founder-market fit, опыт
- **TAM** = $1B+ для Series A
- **Timing** = Why Now? (42% по Gross)

## ЧАСТЬ 6. ДИЗАЙН-МЫШЛЕНИЕ И ИНЖЕНЕРИЯ

### 6.1. Design Thinking (IDEO + Stanford d.school)

**5 этапов:**
```
EMPATHIZE → DEFINE → IDEATE → PROTOTYPE → TEST
     ↑                                       |
     └───────────────────────────────────────┘
```

| Этап | Что делать | Инструмент |
|------|-----------|-----------|
| Empathize | Погрузись в мир пользователя | Интервью, наблюдение, shadowing |
| Define | Сформулируй проблему | Point of View, How Might We |
| Ideate | Генерируй идеи без критики | Brainstorm, Crazy 8's |
| Prototype | Сделай быстрый прототип | Бумага, Figma, код |
| Test | Тестируй с реальными людьми | Usability testing, A/B |

### 6.2. Визуальная архитектура (тренды 2026)

| Тренд | Описание |
|-------|---------|
| **Bold typography** | Гигантские заголовки 44–64pt, sans-serif |
| **Dark mode + neon** | Тёмный фон + яркие акценты |
| **Bento grid** | Сетка в стиле Apple |
| **Animated infographics** | Движение вместо статики |
| **2D animation** | Стилизованная анимация, не гиперреализм |
| **Video-first** | Короткие видео вместо текста |
| **Interactive** | Tap-to-reveal, live polls |
| **1 chart = 1 insight** | Минимализм в данных |

## ЧАСТЬ 8. ПРЕЗЕНТАЦИИ МИРОВОГО УРОВНЯ

### Архитектура идеальной презентации (2026)

**Структура:**
```
1. HOOK (3 сек)       → Крупное число / провокация / вопрос
2. ПРОБЛЕМА           → Кортизол: боль, страх, потери
3. РЕШЕНИЕ            → Эндорфины: облегчение, надежда
4. КАК ЭТО РАБОТАЕТ  → Дофамин: шаги, прогресс
5. ДОКАЗАТЕЛЬСТВА     → Окситоцин: доверие, отзывы, данные
6. РЕЗУЛЬТАТ          → Серотонин: статус, уверенность
7. CTA                → Адреналин: срочность, действие
```

**Визуальные правила (Apple/Stripe уровень):**
- 1 идея = 1 слайд
- Заголовок = вывод (не тема)
- Максимум 3 цвета (1 акцент + 2 нейтральных)
- Шрифт: 44–64pt заголовки, 24–32pt тело
- Анимации 150–300ms, easing: cubic-bezier
- Dark mode + gradient accents
- Интерактивность: стрелки, touch, scroll

**Геймификация:**
- Прогресс-бар (% пройденного)
- Fade-up анимации при появлении контента
- Визуальная иерархия через размер и цвет
- «Переменная награда»: каждый слайд = новый инсайт
