# Промты для Cursor AI — внедрение стратегии в проект

> Скопируй нужный промт и отправь в чат Cursor Agent. Каждый промт = один шаг стратегии.

---

## ШАГ 1. Создание лендинга (MVP)

```
Создай одностраничный лендинг для [описание продукта] на HTML/CSS/JS.

Требования:
- Тёмная тема, крупная типографика (Inter или system-ui)
- Градиентные акценты (фиолетовый → розовый)
- Hero: H1 с главным оффером + подзаголовок + CTA кнопка
- Блок проблемы: 3 карточки с болями клиента
- Блок решения: «было/стало»
- Преимущества: 3–4 карточки с иконками
- Социальное доказательство: отзывы или цифры
- FAQ: аккордеон, 5 вопросов
- CTA повторяется каждые 2 экрана
- Футер с контактами
- Адаптивный (mobile-first)
- Анимации fade-up при скролле (IntersectionObserver)
- SEO: 1 H1, логическая иерархия H2→H3, мета-теги
```

## ШАГ 2. Customer Discovery форма

```
Добавь на лендинг блок для сбора обратной связи:
- Простая форма: имя + email + «Какая ваша главная проблема с [тема]?»
- При отправке показывай «Спасибо! Мы свяжемся в течение 24 часов»
- Данные сохраняй в localStorage (или отправляй на webhook)
- Стиль формы — в едином дизайне с лендингом
```

## ШАГ 3. Pitch Deck презентация

```
Создай pitch deck презентацию в HTML формате для [название стартапа]:

Структура (Sequoia Framework):
1. Purpose — одно предложение о продукте
2. Problem — 3 боли клиента с данными
3. Solution — как мы решаем
4. Why Now — почему сейчас (тренды 2026)
5. Market — TAM/SAM/SOM с цифрами
6. Competition — матрица конкурентов
7. Product — скриншоты/описание
8. Business Model — как зарабатываем
9. Team — основатели
10. Ask — сколько привлекаем

Дизайн:
- Тёмная тема, минимализм
- Навигация стрелками
- Анимации переходов
- Прогресс-бар
- Кнопка скачивания HTML
- Один файл (self-contained)
```

## ШАГ 4. OKR трекер

```
Создай простой OKR трекер на одной HTML странице:
- Ввод Objective (цель) и 3-5 Key Results (ключевые результаты)
- Прогресс-бары для каждого KR (0–100%)
- Цветовая индикация: <30% красный, 30-70% жёлтый, >70% зелёный
- Данные сохраняются в localStorage
- Квартальный таймер обратного отсчёта
- Тёмная тема, минимализм
```

## ШАГ 5. Customer Journey карта

```
Создай интерактивную Customer Journey Map для [продукт]:

Этапы:
1. Осознание проблемы (Awareness) — кортизол
2. Поиск решения (Consideration) — дофамин
3. Оценка (Evaluation) — серотонин
4. Покупка/регистрация (Purchase) — адреналин
5. Использование (Retention) — окситоцин + привычка
6. Рекомендация (Advocacy) — окситоцин + серотонин

Для каждого этапа:
- Эмоция клиента
- Точки контакта
- Метрики
- Действия для улучшения
- Гормон, который активируется

Визуал: горизонтальная timeline, карточки, цветовое кодирование по эмоциям
```

## ШАГ 6. A/B тестирование

```
Добавь систему простого A/B тестирования на лендинг:
- Два варианта заголовка (случайный выбор при загрузке)
- Трекинг кликов по CTA для каждого варианта
- Простая панель статистики (доступна по ?stats в URL)
- Сохранение данных в localStorage
- Показывай конверсию каждого варианта в %
```

## ШАГ 7. Аналитика и метрики

```
Создай dashboard метрик для стартапа (HTML):

Метрики:
- MAU / DAU
- Retention (Day 1, Day 7, Day 30)
- CAC (Customer Acquisition Cost)
- LTV (Lifetime Value)
- LTV/CAC ratio
- MRR (Monthly Recurring Revenue)
- Churn rate
- NPS score

Визуал: карточки с числами, мини-графики (sparklines на canvas)
Тёмная тема, реальное время обновления
Данные — заглушки с возможностью подключения API
```

## ШАГ 8. Конкурентный анализ

```
Создай интерактивную матрицу конкурентного анализа (HTML):
- Таблица: конкуренты по строкам, характеристики по столбцам
- Цветовое кодирование: сильно/средне/слабо
- Наш продукт выделен визуально
- Фильтры по категориям
- Экспорт в PNG (html2canvas)
```

## ШАГ 9. Hook Model Canvas

```
Создай интерактивный Hook Model Canvas (Nir Eyal) для [продукт]:

4 квадранта:
1. TRIGGER (внешний → внутренний)
2. ACTION (минимальное усилие, B=MAP)
3. VARIABLE REWARD (типы: tribe/hunt/self)
4. INVESTMENT (что пользователь вкладывает)

Визуал: 4 карточки в квадрате со стрелками цикла
Редактируемые поля для заполнения
Сохранение в localStorage
```

## ШАГ 10. Полная стратегия в одном файле

```
Создай стратегический документ проекта (HTML):

Разделы:
1. Vision & WHY (Simon Sinek)
2. Problem & JTBD (Christensen)
3. Market: TAM/SAM/SOM
4. Beachhead Market + Wedge
5. Customer Discovery план (Steve Blank)
6. MVP план (Lean Startup)
7. PMF критерии (Sean Ellis 40%)
8. GTM стратегия 2026 (OaaS)
9. Hook Model
10. MOAT стратегия
11. OKR на квартал
12. Финансовая модель (unit economics)
13. Pitch Deck outline
14. Риски + Pre-mortem (стоицизм)

Каждый раздел — редактируемый, сохраняется в localStorage.
Тёмная тема, навигация боковым меню.
```
